What is a Markov Chain

A Markov Chain is a member of a class of probabilistic functions called stochastic processes. A stochastic process is a collection of random variables which trace state and time.

What separates Markov Chains from other stochastic processes is it follows a very simple principle it is memoryless, whatever state or time period your process may be in it does not depend on the past. This can be described mathematically as P(Xn = xn | Xn-1 = xn-1, Xn-2 = xn-2, …, X2 = x2, X1 = x1) = P(Xn = xn | Xn-1 = xn-1). In English this means you want to get to Xn given knowledge of X1 to X2 up to Xn-1 you can get there or alternatively you can get to Xn just by having knowledge of Xn-1.

Modeling baseball

Baseball is well described by a stochastic process because it can be broken up into 25 discrete base out states from bases for example empty 0 out or bases loaded 2 outs all the way up to 3 outs.

Each play transitions whatever the current base out state into a new one. For example if we are in the base out state bases empty and 0 out and someone hits a single then we transition to a man on first and 0 out. Eventually though, every inning must end at 3 outs so we use a special type Markov model with 3 outs as the absorbing state, meaning eventually every path from state to state must end with 3 outs.

The transitions from one base out state to another can vary. Simplistically, if there are none out 0 on a player can reach 5 different base out states by hitting a single/walk, double, triple, homer, or creating an out. Each one of those plays has an associated probability thus our transition from one state to another does too. You can see the the probabilities and base out states represented as a graph below.

Calculations

This graph is the key to our Markov model, transformed into its 25x25 matrix counterpart it is called the Transition Matrix, name it T. By taking Tn as n approaches infinity we can actually see the long term probabilities of transitioning from one state to another. If you multiply Tn by a 1x25 initial state row vector,call it I, which contains the probability of being in any one of the 25 states, we can then see the long term probability of being in any given state. If we take the Expected value of that then we can actually see how many runs are scored in each base out state resulting in the run distribution for any in game situation.

E[ITn] = The Baseball Run Distribution

Using this distribution we can do a couple of useful analyses, such as deterring the value of a bunt or steal and when if that action should be taken by the manager with which player. This distribution can also determine how many runs a team is expected to score in a game or even how many runs a player would produce if they occupied all 9 spots in the lineup

Regression Analysis

One way to check the accuracy of the Markov Model is to see how well it perdicts the average of runs scored in a game. For a baseline test, in orange, we plotted each of the last 90 years against its next and got a fitting line with an R2 of .55 but when we graphed the Markov models run prediction vs the the average runs that year, in green, we got an R2 of .83 which blows away the baseline test.

Flaws

This is a simplistic model, it only incorporates 7 different plays singles, doubles, triples, home runs, walks, and outs. It does not include the probabilities of a stolen bases, sac flies, wild pitches, and errors into the transition matrix. That doesn’t mean its not useful though, it covers the vast majority of plays and still generates a fairly accurate distribution of baseball run scoring.
