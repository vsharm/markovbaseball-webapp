What is a Markov Chain

A Markov Chain is a member of a class of probabilistic functions called stochastic processes. A stochastic process is a collection of random variables which trace state and time.

The memoryless property is what separates Markov Chains from other Stochastic Processes. Each state of a Markov Chain must be completely independent of the past. This can be described mathematically as P(Xn+1 = xn+1 | Xn = xn, Xn-1 = xn-1, …, X2 = x2, X1 = x1) = P(Xn+1 = xn+1 | Xn = xn). Markov Chains trace discrete paths, so the memoryless property requires the next state to be exclusively determined by the current state rather than being dependent on some combination of past states.

Modeling baseball

Baseball is well described by a Markov Chain because it can be broken up into 25 discrete and independent base out states. A few examples are a man on first and 1 out, bases loaded 2 out, or bases empty 0 out. In baseball each play, a hit, out, or walk, transitions the current base out state into a new one. For instance, if there man on first and 1 out and the batter hits a single then the current base out state transitions to a new one, 2 men on base first and second with 1 out.

The transitions from one base out state to another can vary. Simplistically, if the game is in the state bases empty 0 out, a player can reach 5 different base out states via a hit a single/walk/hit by a pitch, double, triple, homer, or out. Each one of those plays has an associated probability thus a transition from one state to another does too.

Each transition from base out state to base out state is solely dependent on the current state. How an inning arived at the bases loaded and 2 outs does not effect what could happen next. The fundamental assumption is that all paths to get to a specific base out state have little to no effect on the transition probabilities to a new base out state. This is the memoryless property, hence why we are modeling baseball using Markov Chains. The base out states are represented in an interactive graph below.

Calculations

This graph is the key to our Markov model, transformed into its 25x25 matrix counterpart it is called the Transition Matrix, denote it as T. Each entry in T is a transition probability from one base out state i to another base out state j. If we take T to a specific power n, for example if n=5 we can observe the probability of being in any base out state in 5 plays. By taking the limit of Tn as n approaches infinity we get the probability distribution of being in any state. Finaly, if we take the expected value of the probabilty distribution then we can actually see how many runs are scored in each base out state, which is the run distribution for any in game situation.

Limn→∞ ( E[Tn] ) = The Baseball Run Distribution

This distribution lends it self to a variety of useful analyses, such as determining when to bunt or steal. It can also find how many runs a team is expected to score in a game, or even the number of runs a player would produce if they occupied all 9 spots in the lineup.

Regression Analysis

One way to check the accuracy of the Markov Model is to see how well it predicts the average runs scored in a game. First, we need a baseline test to compare the Markov Model against, we will use year over year data. So, plotted in orange, is each of the last 90 years of MLB league avg runs per game vs the next years league avg runs per game(think of comparing each yeari to yeari-1). A regression line fits those data with an R2 of .55. In comparison, the Markov models average run prediction was plotted vs the last 90 years of run scoring data, in green, and when regressed it resulted in an R2 of .83 which beats the baseline test.

Flaws

This is a simplistic model, it only incorporates 7 different plays singles, doubles, triples, home runs, walks, hit by pitches, and outs. It does not include the probabilities of a stolen base, sac fly, wild pitch, or error into the transition matrix, but it covers the vast majority of plays and generates a fairly accurate distribution of baseball run scoring.
